Full User Journey

flowchart TD
  A[Welcome] --> B{Authenticated?}
  B -- No --> C[Landing]
  B -- Yes + no profile --> D[Disclaimer]
  D --> E[Health Questionnaire]
  E --> F[Safety Profile Saved]
  F --> C

  C -->|Select Yoga Pose or Sequence| G[Intro]
  G --> H[Framing]
  H --> I[Evaluating]
  I --> J[Results]
  J -->|Next| G
  J -->|Finish sequence| K[Sequence Complete]
  K --> C

  C -->|Select Breathwork| L[Breathwork Session]
  L --> C

Safety-Oriented Session Planning

  1. Frontend submits questionnaire data.
  2. Backend SafetyProfiler computes risk tier and allowed/blocked/modified poses.
  3. SessionOrchestrator builds plan using latest risk profile + optional carry-over risk from previous session.
  4. Plan and warnings are shown in session briefing before practice starts.

Evaluation Loop (Live)

sequenceDiagram
  participant CAM as Camera Stream
  participant FE as App Evaluator Loop
  participant BE as Backend Evaluate

  CAM->>FE: landmarks per frame
  FE->>FE: local framing + stability checks
  FE->>BE: periodic evaluate requests
  BE-->>FE: score + correction + safety_note + credit fields
  FE->>FE: update UI, voice guidance, overlays
  FE->>FE: transition to results on hold or exit
